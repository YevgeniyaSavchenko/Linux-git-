  
УКАЖИТЕ СЛЕДУЮЮЩИЕ КОМАНД:

-для полной очистки индекса:
      git rm

-для просмотра разницы между раб.каталогом и посл. коммитом:
     git diff

-для создания новой ветки и одновременного переключения в на нее:
     git checkout -b

-для сравнения веток между собой:
    git diff ветка1 ветка2
-для удаления локальной ветки:
     git branch -d (delete)
-для слияния веток:
     git merge
-для перезаписи посл/коммита:
     git commit --amend

Коротко опишите причину возникновения конфликтов при слиянии и порядок действий для их решения:

Обычно конфликты возникают, когда два человека изменяют одни и те же строки в файле или один разработчик удаляет файл, который в это время изменяет другой разработчик. В таких случаях Git не может автоматически определить, какое изменение является правильным. Конфликты затрагивают только того разработчика, который выполняет слияние, остальная часть команды о конфликте не знает. Git помечает файл как конфликтующий и останавливает процесс слияния. 

Локальное состояние необходимо стабилизировать с помощью команд git stash, git checkout, git commit или git reset
